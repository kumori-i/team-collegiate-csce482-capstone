ANNEX 7: TEST CASES

Project: CerebroChat
Purpose: This annex contains a complete set of test cases for the testing phases planned for the CerebroChat system.

====================================================================
1. INTRODUCTION
====================================================================

This annex documents the planned and executed test cases for the CerebroChat basketball analytics application. The application includes a React frontend, an Express backend, authentication features, Supabase integration, player search and detail workflows, and AI-assisted chat and scouting functionality.

The test cases are organized by testing phase:

- Phase 1: Smoke Testing
- Phase 2: Functional Testing
- Phase 3: Integration Testing
- Phase 4: User Interface Testing
- Phase 5: Security and Access Control Testing
- Phase 6: AI and Conversation Workflow Testing
- Phase 7: Regression Testing
- Phase 8: User Acceptance Testing Support Cases

====================================================================
2. TEST ENVIRONMENT SUMMARY
====================================================================

Test environment assumptions:

- Frontend: React application running locally
- Backend: Express server running on port 5001
- Database: Supabase project connected and configured
- Browser: Chrome, Edge, Firefox, or Safari
- Authentication: JWT and Google OAuth configuration available
- AI provider: TAMU, Gemini, or Ollama configured depending on environment

Primary local URLs:

- Frontend: React dev server URL
- Backend health: http://localhost:5001/health
- Backend API docs: http://localhost:5001/api/docs

Preconditions: AI provider configured

Expected Result: Response describes the archetype thresholds
Priority: High

Title: Chat identifies a player's archetypes
Objective: Verify player lookup plus archetype resolution
Preconditions: Named player exists in dataset
Steps:
1. Ask "What are Cameron Boozer's archetypes?"
Expected Result: Response lists matching archetype labels
Priority: High

TC-CHAT-005
Title: Chat finds similar players using a named player
Objective: Verify player similarity workflow
Preconditions: Named player exists
Steps:
1. Ask "Find 5 similar players to Cameron Boozer"
Expected Result: Response lists 5 similar players or fewer if the pool is smaller
Priority: High

TC-CHAT-006
Title: Chat finds similar players using archetype-related wording
Objective: Verify similar-player routing is not trapped by archetype explanation flow
Preconditions: Named player exists
Steps:
1. Ask "Find 5 similar players to Cameron Boozer via his archetypes"

TC-CHAT-007

Preconditions: Dataset contains portal-available metadata
Steps:
1. Ask "Find 5 similar players to Cameron Boozer who are active and can be transferred"
Expected Result: Returned players satisfy transfer-availability constraint according to dataset
Priority: High

TC-CHAT-008
Title: Chat can reset session context

Preconditions: Existing conversation in progress
