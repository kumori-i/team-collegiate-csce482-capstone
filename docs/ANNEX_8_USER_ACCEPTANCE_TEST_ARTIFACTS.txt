ANNEX 8: USER ACCEPTANCE TEST ARTIFACTS

Project: CerebroChat
Purpose: This annex contains the complete set of artifacts used in the user acceptance test (UAT) process for the CerebroChat system.

====================================================================
1. INTRODUCTION
====================================================================

User Acceptance Testing was performed to confirm that CerebroChat meets the needs of its intended users. The purpose of UAT was to validate that the system is usable, understandable, and functionally appropriate for basketball-focused workflows such as player search, scouting review, archetype interpretation, and chat-based data queries.

This annex collects the artifacts that support the UAT process, including:

- UAT plan
- participant profile form
- test session setup
- UAT scenarios
- task script
- observation sheet
- feedback form
- issue log
- results summary
- sign-off template

====================================================================
2. UAT PLAN
====================================================================

2.1 UAT Objective

The objective of UAT was to determine whether target users could successfully operate the CerebroChat system and complete key tasks without developer intervention.

2.2 UAT Scope

Included in UAT:

- login and access
- navigation
- player search
- player detail review
- archetype interpretation
- AI chat usage
- similar-player requests
- report generation

Not emphasized in UAT:

- backend code-level diagnostics
- stress testing
- low-level API schema validation

2.3 UAT Success Criteria

The system is considered acceptable if:

- users can complete core tasks successfully,
- the interface is understandable,
- generated outputs are useful to the target user,
- major blockers are not present,
- participants report that the system supports intended basketball analysis tasks.

5. UAT TASK SCENARIOS
====================================================================

Artifact Name: UAT Scenario Set

Scenario UAT-01
Title: User signs in and reaches the home page
Goal: Confirm that the user can access the system
Expected Outcome: User reaches the authenticated home page

Scenario UAT-02
Title: User searches for a player
Goal: Confirm that a user can locate player records
Expected Outcome: Relevant player search results are returned

Scenario UAT-03
Title: User opens a player details page
Goal: Confirm that the user can inspect player details
Expected Outcome: Player metrics, charts, and archetypes are displayed

Scenario UAT-04
Title: User identifies a player's archetypes
Goal: Confirm that archetype information is understandable and available
Expected Outcome: User can view or ask for archetypes successfully

Scenario UAT-05
Title: User asks the chat to explain an archetype
Goal: Confirm that the assistant can explain archetype meaning
Expected Outcome: Chat returns a useful definition of the archetype

Scenario UAT-06
Title: User asks for similar players
Goal: Confirm that conversational similarity workflow is usable
Expected Outcome: Chat returns a list of similar players

Scenario UAT-07
Title: User asks for transferable/portal-available similar players
Goal: Confirm that transfer-related language is correctly handled
Expected Outcome: Returned players satisfy the transfer-availability intent

Scenario UAT-08
Title: User generates a scouting report
Goal: Confirm that report generation is useful and understandable
Expected Outcome: A readable report is produced with useful player information

Scenario UAT-09
Title: User resets chat and starts over
Goal: Confirm that session reset works
Expected Outcome: Prior context does not interfere with the next task

6. UAT TASK SCRIPT
====================================================================

Artifact Name: Facilitator Task Script

Instructions to participant:

1. Sign in to the system.
2. Navigate to the Search page.
3. Search for a player by name.
4. Open the player's details page.
5. Review the listed archetypes.
6. Open the Chat page.
7. Ask, "What is a Modern Big?"
8. Ask for a specific player's archetypes.
9. Ask for similar players to that player.
10. Ask for similar players who are transferable or in the transfer portal.
11. Return to the player details page and generate a scouting report.
12. Reset the chat and log out.

Facilitator notes:

- Do not assist unless the participant is blocked.
- Record hesitation, confusion, and misinterpretation.
- Record whether the participant completes each task independently.
